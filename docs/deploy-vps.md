# Deploying a WebsiteEmpire website to a Contabo VPS

## Prerequisites
- A published deploy/ folder in your working directory (generated by WebsiteEmpire → Publish)
- A domain name registered on Namecheap
- A Contabo VPS (Cloud VPS 10 NVMe, Ubuntu 24.04)

---

## Step 1 — Point domain DNS to VPS (Namecheap)

1. Log into Namecheap → **Domain List** → **Manage** on your domain
2. Go to **Advanced DNS** tab
3. Delete any existing A records
4. Add two A records:

| Type     | Host | Value         | TTL       |
|----------|------|---------------|-----------|
| A Record | `@`  | YOUR_VPS_IP   | Automatic |
| A Record | `www`| YOUR_VPS_IP   | Automatic |

Leave **Nameservers** unchanged (keep Namecheap BasicDNS).

DNS propagates in 5 minutes to 2 hours. Test with:
```bash
nslookup yourdomain.com
```

---

## Step 2 — Prepare the VPS

```bash
ssh root@YOUR_VPS_IP

apt update && apt upgrade -y
apt install -y libjsoncpp25 nginx certbot python3-certbot-nginx ufw sqlite3

ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw enable

mkdir -p /opt/websiteempire/deploy
```

---

## Step 3 — Upload the binary and website files

Run from your dev machine:

```bash
# Upload the Drogon server binary
scp /home/cedric/Applications/WebsiteEmpire2/build/StaticWebsiteServe/StaticWebsiteServe \
    root@YOUR_VPS_IP:/opt/websiteempire/

# Upload the deploy directory (content.db per language + shared images.db)
rsync -avz --progress \
    /path/to/workingDirectory/deploy/ \
    root@YOUR_VPS_IP:/opt/websiteempire/deploy/
```

The deploy/ folder structure:
```
deploy/
  images.db        ← shared across all languages
  en/
    content.db     ← English pages
  fr/
    content.db     ← French pages
```

---

## Step 4 — Start the Drogon servers

```bash
ssh root@YOUR_VPS_IP

chmod +x /opt/websiteempire/StaticWebsiteServe

# Start English server (port 8080)
cd /opt/websiteempire/deploy/en
/opt/websiteempire/StaticWebsiteServe --port 8080 --lang en \
    --images-db /opt/websiteempire/deploy/images.db &

# Start French server (port 8081)
cd /opt/websiteempire/deploy/fr
/opt/websiteempire/StaticWebsiteServe --port 8081 --lang fr \
    --images-db /opt/websiteempire/deploy/images.db &

# Test (Drogon serves gzip — use --compressed)
curl -s --compressed http://localhost:8080/index.html | head -5
curl -s --compressed http://localhost:8081/fr/ | head -5
```

---

## Step 5 — Configure nginx

```bash
cat > /etc/nginx/sites-available/yourdomain.com << 'EOF'
server {
    server_name yourdomain.com www.yourdomain.com;

    # French (must be above the catch-all)
    location /fr/ {
        proxy_pass http://127.0.0.1:8081;
        proxy_set_header Host $host;
    }

    # English (catch-all — always last)
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
    }
}
EOF

ln -s /etc/nginx/sites-available/yourdomain.com /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

Add more `location /langcode/` blocks above `location /` for each additional language.

---

## Step 6 — SSL (free, auto-renewing)

```bash
certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

- Enter your email when asked (used for expiry reminders only, not public)
- Accept the Terms of Service
- Certificate is valid 90 days and **renews automatically** — no action needed

---

## Step 7 — Set up systemd (auto-start on reboot)

```bash
cat > /etc/systemd/system/website-en.service << 'EOF'
[Unit]
Description=WebsiteEmpire EN
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/websiteempire/deploy/en
ExecStart=/opt/websiteempire/StaticWebsiteServe --port 8080 --lang en --images-db /opt/websiteempire/deploy/images.db
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

cat > /etc/systemd/system/website-fr.service << 'EOF'
[Unit]
Description=WebsiteEmpire FR
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/websiteempire/deploy/fr
ExecStart=/opt/websiteempire/StaticWebsiteServe --port 8081 --lang fr --images-db /opt/websiteempire/deploy/images.db
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now website-en website-fr website-de website-ja
systemctl status website-en website-fr website-de website-ja
```

## Redeploying after a republish

When you republish from WebsiteEmpire, sync the updated files, verify integrity, then restart.
**Important:** wait for WebsiteEmpire to finish publishing before running rsync — copying a
SQLite file while it is still being written produces a malformed image on the VPS.

```bash
rsync -avz --progress \
    /path/to/workingDirectory/deploy/ \
    root@YOUR_VPS_IP:/opt/websiteempire/deploy/

# Verify each content.db arrived intact before restarting
ssh root@YOUR_VPS_IP "for lang in en fr de ja; do \
    echo \"=== \$lang ===\"; \
    sqlite3 /opt/websiteempire/deploy/\$lang/content.db 'PRAGMA integrity_check; SELECT COUNT(*) FROM pages;'; \
done"

ssh root@YOUR_VPS_IP "systemctl restart website-en website-fr website-de website-ja"
```

If `PRAGMA integrity_check` returns anything other than `ok`, re-sync that language individually:

```bash
rsync -avz --progress --checksum \
    /path/to/workingDirectory/deploy/de/content.db \
    root@YOUR_VPS_IP:/opt/websiteempire/deploy/de/content.db
ssh root@YOUR_VPS_IP "systemctl restart website-de"
```

## Useful commands on the VPS

```bash
# Check servers are running
systemctl status website-en website-fr website-de website-ja

# View logs
journalctl -u website-en -f
journalctl -u website-fr -f
journalctl -u website-de -f
journalctl -u website-ja -f

# Check a page is served correctly
curl -s --compressed https://yourdomain.com/index.html | head -5

# Check SSL renewal status
certbot renew --dry-run
```
